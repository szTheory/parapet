# Roadmap: v0.6 Change Correlation & Audit Trailing

## Milestone Goal
**"Every alert is correlated to the change that caused it."**

This milestone focuses on reducing Time-To-Resolution (TTR) by automatically correlating SLO burn rates with `Rulestead` feature flag toggles, extracting OpenTelemetry trace exemplars for direct metric-to-trace linking, and ensuring compliance parity via `Threadline`.

## Phases

### Phase 1: OpenTelemetry Trace Exemplars
**Goal:** Operators can click on an anomalous metric in Grafana or an Incident in the UI and jump directly to the offending trace.
**Requirements:** OTL-01
**Plans:**
- [x] 01-01-PLAN.md — Implement OTel trace_id extraction from telemetry events and process dictionary
- [ ] 01-02-PLAN.md — Append trace_ids as Prometheus exemplars in metric output
- [ ] 01-03-PLAN.md — Store trace_id on Incident schemas and surface dynamic trace links in Operator UI

### Phase 2: Rulestead Flag Correlation
**Goal:** If a feature flag causes a regression, it is immediately obvious on the incident dashboard.
**Requirements:** RUL-01, RUL-02
**Plans:**
- [ ] 02-01-PLAN.md — Implement Parapet.Integrations.Rulestead adapter to consume flag telemetry
- [ ] 02-02-PLAN.md — Emit flag change metrics and record durable timeline entries for global flag flips
- [ ] 02-03-PLAN.md — Update Operator UI to correlate and highlight proximate Rulestead flag changes on Incident pages

### Phase 3: Threadline Compliance Sync
**Goal:** Guarantee that all Parapet operator actions (like executing runbooks) satisfy strict compliance requirements by mirroring to Threadline.
**Requirements:** THR-01
**Plans:**
- [ ] 03-01-PLAN.md — Implement Parapet.Integrations.Threadline adapter for audit sync
- [ ] 03-02-PLAN.md — Update `ToolAudit` module to support dual-write or Threadline-deferred modes
- [ ] 03-03-PLAN.md — Ensure 100% test coverage and verify optional dependency "compile out" isolation
