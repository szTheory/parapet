# Plan: 04-02-PLAN.md

## Goal
Build an Automated Retrospective generator that compiles incident context, timelines, and mitigation actions into a unified Markdown document.

## Context
When an incident is resolved, operators often need to assemble a post-mortem or retrospective. Parapet already holds all the durable evidence (the incident metadata, timeline entries, tool audits, notifications). We will provide a function to render this structured data into a digestible Markdown format, heavily reducing the toil of incident review.

## Tasks

1. **Create `Parapet.Evidence.Retrospective`:**
   - Create `lib/parapet/evidence/retrospective.ex`.
   - Implement `generate_markdown(incident_id)` which:
     - Fetches the `Incident` and preloads its `TimelineEntry` records (ordered by time).
     - Formats a Markdown document containing:
       - Incident Title & Description
       - Time to Acknowledge / Time to Resolve metrics
       - A chronological log of timeline events (Alerts, Mitigations, Notifications).

2. **Integrate Retrospective Generation:**
   - Extend `Parapet.Operator.resolve_incident/2` to automatically attach the generated retrospective to the incident record (e.g., in the `runbook_data` map or as a final timeline entry with type `narrative_edit` or a new type `:retrospective`).

3. **Extend Operator UI (`parapet.gen.ui`):**
   - In `priv/templates/parapet.gen.ui/operator_detail_live.ex.eex`, add a section visible when an incident is `resolved` that displays the "Automated Retrospective" content in a `<pre><code>` block or Markdown renderer.
   - Provide a "Copy to Clipboard" capability for the retrospective text.

4. **Update Tests:**
   - Add tests for `Parapet.Evidence.Retrospective` to assert formatting and data inclusion.
   - Add tests ensuring `resolve_incident/2` triggers retrospective generation.

## Verification
- Ensure `mix test` passes.
- Trigger an incident resolution in a test/sandbox environment and confirm the retrospective is correctly compiled and displayed in the generated LiveView.
