<objective>
Create the `Parapet.Notifier` behavior, implement broadcasting to adapters, and wire into the incident lifecycle.

Purpose: This establishes the foundation for asynchronous incident notifications, ensuring we do not block the web server, durably record all notification attempts to the incident timeline, and trigger them automatically.
Output: The `@behaviour` contract, `broadcast/1` dispatching mechanism, `Oban` integration (falling back to `Task`), and triggering from `AlertProcessor`.
</objective>

    Create `lib/parapet/notifier.ex` and define `Parapet.Notifier` with a `@callback deliver(incident :: Parapet.Spine.Incident.t(), opts :: keyword()) :: {:ok, term()} | {:error, term()}`. 
    Implement a `broadcast(incident)` function that reads configured notifiers via `Application.get_env(:parapet, :notifiers, [])`. Iterate through each configured adapter and call `dispatch(incident, adapter, opts)`.
    The `dispatch/3` logic must check if `:oban` is loaded via `Code.ensure_loaded?(Oban)`. 
    If Oban is present, enqueue a job to `Parapet.Notifier.ObanWorker` (created in Task 2). If Oban is absent, fallback to `Task.start/1` to execute the notification asynchronously. Write tests verifying this behavior.
  </action>
  <verify>
    <automated>mix deps.get && mix test test/parapet/notifier_test.exs</automated>
  </verify>
  <done>Req is an optional dependency. Notifier exposes broadcast/1 which dispatches to all configured adapters dynamically without blocking.</done>
</task>
