# Requirements: Parapet v0.2

## Categories

- [SPINE] Durable Evidence Spine
- [UI] Operator Interface
- [ECO] Ecosystem Integrations

## Requirements

### Durable Evidence Spine (SPINE)
- SPINE-01: System provides Ecto schemas for Incidents with a state machine (open, investigating, resolved).
- SPINE-02: System provides Ecto schemas for Timeline Entries linked to Incidents to durably track alerts, notes, and actions.
- SPINE-03: System provides Ecto schemas for Tool Audits to securely log AI and human MCP tool calls.
- SPINE-04: System enforces a clear boundary that prevents raw high-volume telemetry from being written to Ecto to avoid database saturation.

### Operator Interface (UI)
- UI-01: System provides a Phoenix LiveView SRE dashboard for operators to view and manage open incidents and their timelines.
- UI-02: System provides a secure UI surface to execute and audit safe application mutations (e.g., toggling feature flags, viewing runbooks).
- UI-03: System UI integrates external visualization links (e.g., Grafana) and simple text readouts rather than rebuilding complex time-series charting in LiveView.
- UI-04: System provides generators and documentation ensuring the LiveView UI is secured behind authentication in host applications.

### Ecosystem Integrations (ECO)
- ECO-01: System provides an optional integration adapter for Rulestead to track feature flag changes and enable flag-toggling mitigations via the Operator UI.
- ECO-02: System provides optional integration adapters for Mailglass and Chimeway to establish out-of-the-box deliverability SLIs.
- ECO-03: System provides optional integration adapters for Accrue (billing) and Rindle (media processing) to track business-specific journey health.
- ECO-04: System strictly adheres to the "compile out cleanly" constraint, ensuring core functionality runs without any sibling libraries installed using dynamic module checks.
- ECO-05: System conceptually integrates with Threadline for durable audit history interoperability.
