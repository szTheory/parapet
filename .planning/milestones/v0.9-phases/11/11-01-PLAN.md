## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local test harness -> distributed runtime | the canary crosses from normal ExUnit execution into environment-dependent BEAM distribution startup |
| unsupported environment -> proof report | a fragile startup path could be misreported as a passed peer-node proof |
| proof-only test changes -> product semantics | test harness changes must not leak into runtime guarantees or doctor behavior |

| T-11-02 | R | peer canary verification path | mitigate | Make unsupported environments exit through an explicit branch that states peer-node behavior was not exercised and keeps the DB-backed contention suite as the authoritative proof. |
| T-11-03 | T | `test/support/concurrency_case.ex` helper | mitigate | Scope the new helper to test-only support and preserve the existing sandbox/repo harness so runtime code and doctor semantics remain unchanged. |
| T-11-04 | E | product/runtime posture | accept | This plan intentionally does not touch runtime modules or public tasks; no new runtime authority is introduced. |
</threat_model>
