<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL queue params -> Ecto query builder | User-controlled queue navigation must not create oversized or malformed queries |
| Incident evidence -> queue row projection | Durable evidence must be condensed without leaking high-cardinality or detail-only data into the queue |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Denial of Service | Queue param parsing and page size | mitigate | Clamp page size, validate cursor shape and direction, and fall back to the first page on invalid input. |
| T-03-02 | Tampering | Keyset ordering / cursor boundaries | mitigate | Use exact `updated_at` + `id` deterministic ordering so page boundaries cannot duplicate or skip rows unpredictably under ties. |
| T-03-03 | Denial of Service | Incident query path / index alignment | mitigate | Ship explicit partial indexes for active and resolved browsing so the new queue contract does not rely on table scans at scale. |
</threat_model>
