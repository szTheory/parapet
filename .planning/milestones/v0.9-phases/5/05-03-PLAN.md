<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Escalation worker -> external policy adapter | A retry or competing worker must not duplicate the same external alert |
| Operator-written runbook state -> worker claim transaction | Manual trigger and suppression metadata must be re-checked against live DB state |
| Static doctor source scan -> maintainer trust | Advisory checks must not overclaim distributed correctness |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-07 | Repudiation | Escalation alert chronology | mitigate | Persist explicit executed, conflicted, short-circuited, and retryable outcome types. |
| T-05-08 | Denial of Service | Retry storm after crash | mitigate | Resume from durable claim state with the same idempotency key instead of re-dispatching blindly. |
| T-05-09 | Spoofing | Static cluster assurances | mitigate | Keep `mix parapet.doctor cluster_static` advisory-only and explicitly defer correctness proof to real tests. |
</threat_model>
