  <action>
Implement `Mix.Tasks.Verify.PublicApi`.
The task must:
1. `Code.compile_file` or load all application modules. (E.g., `{:ok, modules} = application:get_key(:parapet, :modules)`)
2. Iterate through modules starting with `Parapet.` but strictly EXCLUDING modules that start with `Parapet.Internal.`.
3. For each public module, call `Code.fetch_docs(module)`. Determine if documentation is present or missing (`{:error, _}`, `{:docs_v1, _, :elixir, _, :hidden, _, _}` or no docs).
4. Generate a structured machine-readable manifest (e.g. JSON string via `Jason.encode!` if Jason is available, or structured Erlang map formatting) that lists all public modules and their documentation status. Output this manifest to stdout.
5. If any public module is missing docs, call `System.halt(1)` after printing the manifest.
  </action>
  <acceptance_criteria>
    - `lib/mix/tasks/verify.public_api.ex` contains exact string `Code.fetch_docs`
    - `lib/mix/tasks/verify.public_api.ex` contains exact string `System.halt(1)`
    - `lib/mix/tasks/verify.public_api.ex` contains exact string `Parapet.Internal.` (for exclusion)
    - Task explicitly builds and outputs a structured manifest (like JSON or map format) of the API surface.
  </acceptance_criteria>
  <verify>
    <automated>grep "Code.fetch_docs" lib/mix/tasks/verify.public_api.ex</automated>
  </verify>
  <done>Verification task generates structured manifest and is built for CI.</done>
