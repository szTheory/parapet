# Plan: 04-01 - Prometheus Artifact Generator

## Goal
Implement `mix parapet.gen.prometheus` to generate valid Prometheus recording and alerting rules based on the user's defined SLOs. (Addresses PROM-01, PROM-02, PROM-03, PROM-04).

## Approach
1. **Create Generator Task:** Add `lib/mix/tasks/parapet.gen.prometheus.ex`.
2. **EEx Templates:** Create EEx templates for the Prometheus YAML rules inside `priv/templates/parapet.gen.prometheus/`. We need templates that output the multi-window burn rate math required by `SLO-02`, `SLO-03`, and `SLO-04`.
3. **Discover SLOs:** The task needs to discover the SLO definitions. We'll need a mechanism (like `Parapet.SLO.list_all/0` or requiring the user to pass a module containing their SLOs, or perhaps just generating a *default* set of rules for HTTP/Oban/Login based on the integrations detected). Given the SRE research, we should generate the "starter set". Let's look at `Parapet.SLO.to_prometheus_rules/1` defined in Phase 3. The generator should call this function for the built-in SLOs (HTTP, Oban, Login).
4. **File Writing:** Write the generated YAML to `priv/parapet/prometheus/rules.yml`.
5. **Validation/Safety:** Use Igniter to handle the file writing to ensure we don't blindly overwrite user changes without a `--force` flag (or handle it manually with standard Elixir `File.exists?` and prompts). Igniter's `create_new_file` or similar might be too strict if it already exists, so we'll need to check the file contents.
6. **Integration Test:** Write a test that runs the generator and then shells out to `promtool check rules` to guarantee PROM-02.

## Success Criteria
- `mix parapet.gen.prometheus` creates `priv/parapet/prometheus/rules.yml`.
- The generated YAML includes recording and alerting rules for HTTP, Oban (if present), and Login (if present).
- `promtool check rules` passes on the generated file.
- Re-running warns before overwriting.
