<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LiveView/UI caller -> `Parapet.Operator` | Untrusted UI input crosses into durable incident and audit writes. |
| `Parapet.Operator` -> `Parapet.Evidence` | Mutating commands cross from validated operator intent into durable storage. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-00 | Tampering | `Parapet.Operator.WorkbenchContract` | mitigate | Encode an explicit derivation contract for workbench-only fields so UI consumers cannot invent severity, approval, or recommendation state ad hoc from incomplete payloads. |
| T-02-01 | Spoofing | `Parapet.Operator.ActionPayload` | mitigate | Require explicit `actor` and `correlation_id` fields and reject blank values before any write path executes. |
| T-02-02 | Tampering | `Parapet.Operator` | mitigate | Encode first-class commands only; do not expose a generic action runner, and route every mutation through the `Ecto.Multi` evidence seam, per D-07 through D-09 and D-17. |
| T-02-03 | Repudiation | `Parapet.Operator` | mitigate | Couple every mutating command to reason/correlation metadata and the corresponding timeline/audit semantics inside one transaction per D-17 and D-18. |
| T-02-04 | Information Disclosure | incident detail payload | mitigate | Keep tool-audit exposure deliberate and summary-oriented; never treat audit records as raw debug dumps per D-19. |
| T-02-05 | Elevation of Privilege | operator narrative editing | mitigate | Distinguish editable narrative fields from immutable factual records so later UI layers cannot edit state transitions or tool actions per D-10 through D-12. |
</threat_model>
