<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Generator -> host router/auth scope | Generated code crosses into host security configuration that Parapet must not silently own. |
| Generated LiveView -> `Parapet.Operator` | UI input is transformed into durable reads and audited commands. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Elevation of Privilege | `mix parapet.gen.ui` | mitigate | Emit only authenticated-scope router guidance and never create Parapet-owned auth plugs or `live_session` policy, per D-13 through D-16. |
| T-02-07 | Tampering | generated action rail | mitigate | Render only first-class action forms tied to `Parapet.Operator` commands, not arbitrary mutation runners, per D-07 through D-09. |
| T-02-08 | Information Disclosure | generated incident workbench | mitigate | Keep tool-audit payloads collapsed/summary-first and keep Grafana/runbook content as links, not embedded raw dumps or charts, per D-05 and D-19. |
| T-02-09 | Denial of Service | generated UI scope | accept | LiveView UI is intentionally generated into host apps only after opt-in; no runtime route is added unless the host chooses to mount it. |
</threat_model>
