<objective>
Append trace_ids as Prometheus exemplars in metric output.

Purpose: Operators need to link directly from metrics in Grafana to traces. Since standard Prometheus Exporters in Elixir don't support OpenMetrics exemplars natively, we must build a custom formatter and capture trace IDs from telemetry events.
Output: A custom Prometheus formatter that appends `# {trace_id="..."}` exemplars to the scrape output using an in-memory buffer, and the telemetry wiring to populate that buffer.
</objective>
