> and were verified by the v0.9 milestone audit (status: `passed`, 12/12 requirements,
> 12/12 phases, 7/7 integration, 8/8 flows). See `v0.9-MILESTONE-AUDIT.md` for the

## Overview
Milestone v0.9 shifted focus from feature breadth to operational depth. With the core
SRE primitives, Operator UI, and integrations in place, the system proved it can handle
high scale without degrading the host application: protecting the TSDB from cardinality
explosions, protecting the Postgres DB from evidence bloat, and streamlining the
developer experience for new adoptions.

## System Requirements

### PERF-01: TSDB Cardinality Protection
- [x] System provides a `mix parapet.doctor cardinality` sub-command to statically analyze metrics configurations and flag unsafe label patterns. — **Validated** (Phase 1/6)
- [x] System strictly limits the number of labels per metric at compile-time to prevent accidental TSDB explosion. — **Validated** (max 10 labels/metric via `Parapet.Metrics.Validator`)

### SCALE-01: Database Pruning & Indexing
- [x] System provides optimized Ecto migrations to add composite indexes to `Incident`, `TimelineEntry`, and `ToolAudit` for fast querying at >100k rows. — **Validated** (Phase 2)
- [x] System provides a `Parapet.Evidence.Archiver` module and `mix parapet.archive` task to safely soft-delete or export resolved incidents older than a configurable window. — **Validated** (Phase 2/10; resolved-only retention)
- [x] Operator UI Incident list utilizes efficient pagination or cursor-based scrolling to prevent large payload rendering issues. — **Validated** (Phase 3/7/14; bounded queue paging)

### DX-01: Unified Install Path
- [x] System provides `mix parapet.install` as a unified, interactive starting point that sequentially runs necessary sub-generators. — **Validated** (Phase 4/8)
- [x] System's `mix parapet.doctor` checks for correct multi-node configuration (e.g., verifying Oban uniqueness settings for escalations). — **Validated** (Phase 4/8)

### SCALE-02: Multi-Node Consistency
- [x] System test suite includes multi-node or concurrency simulation tests verifying that Ecto-backed circuit breakers prevent race conditions when multiple nodes attempt auto-mitigation simultaneously. — **Validated** (Phase 5/11; environment-conditional peer canary)

## Acceptance Criteria
- [x] A developer can run `mix parapet.install` and get the spine and default Prometheus artifacts in one guided flow, with the optional operator UI offered explicitly when LiveView is present. — **Validated** (AC-01, Phase 4/8)
- [x] Running `mix parapet.archive --days 90` successfully moves/clears old evidence without violating foreign key constraints while leaving active `investigating` work untouched. — **Validated** (AC-02, Phase 10)
- [x] The Operator UI loads instantly with 50,000 generated incident records, proving pagination and index effectiveness. — **Validated** (AC-03, Phase 3/14; benchmark `dataset.total_incidents=50120`, `queue.visible_rows=30`)

## Notes on Outcomes

- **Two requirements required closure repair after the initial audit:** archive retention
  semantics (`SCALE-01.b`/`AC-02`, repaired in Phase 10) and the generated operator
  resolve flow (`SCALE-01.c`/`AC-03`, repaired in Phase 13 and backstopped in Phase 14).
- **`SCALE-02` peer-node verification is environment-conditional:** the DB-backed
  contention proof is canonical; the distributed-Erlang peer canary skips cleanly when
  distributed Erlang is unavailable (Phase 11).
