<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| source/config analysis -> doctor findings | static analysis turns host code and config into safety claims that must stay honest about certainty |
| runtime doctor mode -> loaded app state | live probes cross from Mix into application/runtime state and may fail independently of actual findings |
| CI automation -> exit codes | downstream automation trusts exit codes to distinguish ordinary findings from broken doctor execution |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | I | `lib/mix/tasks/parapet.doctor.ex` | mitigate | Make static multi-node checks say they cannot prove distributed correctness and classify ambiguity as `warn` or `skip`, per D-21 and D-23. |
| T-04-06 | D | runtime doctor mode | mitigate | Reserve exit code `2` for doctor/probe execution failures so CI can distinguish broken probing from normal findings, per D-20 and D-22. |
| T-04-07 | T | Oban uniqueness posture checks | mitigate | Treat missing required escalation uniqueness / contradictory config as `error` and cover with fixtures in `test/mix/tasks/parapet.doctor_test.exs`, per D-23. |
| T-04-08 | R | machine-readable doctor output | mitigate | Preserve stable JSON per-check keys and explicit threshold semantics so automation logs can prove why a run failed, per D-18 through D-20. |
</threat_model>
