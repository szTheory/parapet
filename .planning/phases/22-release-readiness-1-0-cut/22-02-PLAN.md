<objective>
Extend the existing Release Please workflow so a created release automatically publishes to
Hex.pm using the locked sequence `mix hex.publish --dry-run` -> `mix hex.publish --yes` ->
post-publish verify, while preserving Release Please as the sole owner of release/tag
creation.

Purpose: REL-02 is currently missing entirely; the repo has Release Please but no publish job.

Output: `.github/workflows/release-please.yml` contains a second job that runs only after a
real Release Please release event and performs the full publish/verify sequence.
</objective>

  <files>.github/workflows/release-please.yml</files>
  <action>
Edit `.github/workflows/release-please.yml` so it:

1. Preserves the existing `release-please` job as the release-creation step.
2. Exposes any outputs needed from that job to downstream jobs.
3. Adds a `publish-hex` job that:
   - `needs: [release-please]`
   - runs only when Release Please reports a created release
   - sets up Elixir/OTP consistently with CI
   - installs dependencies
   - runs `mix hex.publish --dry-run`
   - runs `mix hex.publish --yes`
   - runs explicit post-publish verification commands for both Hex and HexDocs resolution
   - uses `HEX_API_KEY` from repo secrets

Do not add a publish path that runs on ordinary pushes. Do not replace Release Please.
  </action>
  <verify>
    <automated>python3 -c "import yaml; yaml.safe_load(open('.github/workflows/release-please.yml'))" && rg -n "release_created|hex.publish --dry-run|hex.publish --yes|HEX_API_KEY" .github/workflows/release-please.yml</automated>
  </verify>

    - `HEX_API_KEY` is the configured secret name
  </acceptance_criteria>
