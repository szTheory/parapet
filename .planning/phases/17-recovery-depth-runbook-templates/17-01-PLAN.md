<objective>
Land the `warning:` three-surface foundation so a step-level warning actually renders end-to-end, and gate it with two regression-test layers (schema + projection). This plan MUST complete before any template references `warning:` — Elixir silently swallows unknown macro keyword args, so a template using `warning:` before this change compiles cleanly and passes schema tests yet never shows the warning (D-01, sequencing constraint).

Purpose: `warning:` is currently dropped at two silent layers — the `step/2` macro builds the step map from exactly 11 keys (no `warning`), and `WorkbenchContract.derive_runbook_steps/3` re-projects into an explicit field list (no `warning`). Both must be threaded, plus a UI render block, plus a `@doc` so `mix verify.public_api` (`mix docs --warnings-as-errors`) does not break CI (D-12).

Output: A working `warning:` DSL surface across macro, projection, and UI; documented `step/2`; and two failing-then-passing regression tests proving `warning` survives the schema and projection layers.
</objective>
