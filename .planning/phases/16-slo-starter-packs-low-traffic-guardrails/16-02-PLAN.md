<objective>
Ship `Parapet.SLO.StarterPack.DeliverySaaS` — a new `@behaviour Parapet.SLO.Provider` module that
composes the three WebSaaS slices PLUS the Mailglass and Chimeway delivery slices, where the delivery
slices register only when the corresponding host library is loaded and compile out cleanly otherwise.
Delivery slices are DELEGATED from the existing `MailglassDelivery`/`ChimewayDelivery` catalogs (no
inline redefinition → no objective drift). This is the code surface SLO-02 requires.

Purpose: Let a delivery-sending adopter extend the WebSaaS starter set in one line
(`config :parapet, providers: [Parapet.SLO.StarterPack.DeliverySaaS]`) and get coherent delivery SLOs
without configuring anything, while teams that do not send mail/notifications are not burdened with
delivery rules.

Output: `lib/parapet/slo/starter_pack/delivery_saas.ex` + `test/parapet/slo/starter_pack/delivery_saas_test.exs`.
</objective>
