<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| adopter config → Parapet SLO engine | `config :parapet, providers: [...]` is host-controlled; no untrusted network input. SliceSpec construction validates kind/alert_class/matchers at compile/load time. |
| pack SliceSpec → Prometheus TSDB | matcher/group-label keys become PromQL label selectors; high-cardinality keys would blow up TSDB. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-01 | Spoofing (false confidence) | WebSaaS slice `*_source_metric` | mitigate | Pin metric names to REAL emitters — `parapet_http_request_count` / `parapet_journey_login_count` / `parapet_oban_jobs_total` (Task 2 acceptance grep forbids `_duration_milliseconds_count`); legacy `slo/*.ex` names produce dead rules and are explicitly excluded. |
| T-16-02 | Elevation (TSDB cardinality blowup) | WebSaaS slice matcher / group_labels keys | mitigate | `LabelPolicy.assert_safe!` test over every slice's matcher + group-label keys (Task 1 behavior); no `route`/`id`/`path`/`token` keys; HTTP matches `status_class`+`method` only (D-11). |
| T-16-03 | Denial (alert flapping on low traffic) | WebSaaS slice `min_total_rate` | mitigate | Non-zero `min_total_rate` (default 0.01) on every slice; registration test asserts the generated alert carries the `> 0.01` denominator guard (D-10). |
| T-16-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs in this plan — pure in-repo Elixir module + test addition; no new deps (milestone constraint: no new runtime deps). |
</threat_model>
