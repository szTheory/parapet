# Plan: 04-02 - Grafana Artifact Generator

## Goal
Implement `mix parapet.gen.grafana` to write importable Grafana dashboard JSON and provisioning YAML. (Addresses GRAF-01, GRAF-02, GRAF-03, GRAF-04).

## Approach
1. **Create Generator Task:** Add `lib/mix/tasks/parapet.gen.grafana.ex`.
2. **EEx Templates / Static Assets:** Since Grafana JSON is massive and complex, writing an EEx template from scratch is error-prone. We will bundle pre-built, high-quality Grafana JSON dashboards (e.g., an "Executive Health" dashboard, a "RED Services" dashboard, and an "Oban Jobs" dashboard) as static assets or EEx templates inside `priv/templates/parapet.gen.grafana/`.
3. **Dynamic Elements:** The templates must dynamically replace things like the PromQL datasource UID (usually just `${DS_PROMETHEUS}` or a configured variable) and perhaps the app name.
4. **Deploy Markers:** Ensure the dashboard JSON includes an Annotations query that looks for `parapet_deploy_info` (satisfying GRAF-04).
5. **Provisioning YAML:** Generate a `dashboards.yml` provisioning file so adopters can drop it into their Grafana setup.
6. **File Writing:** Write to `priv/parapet/grafana/dashboards/*.json` and `priv/parapet/grafana/provisioning/dashboards.yml`.

## Success Criteria
- `mix parapet.gen.grafana` creates JSON dashboards and a provisioning YAML in `priv/parapet/grafana/`.
- The dashboards include panels for HTTP RED metrics, Oban queues, SLO burn rates, and Deploy markers.
- Dashboards use the recording rules generated by 04-01, not raw complex PromQL in the panels.
