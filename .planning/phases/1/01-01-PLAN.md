<objective>
Implement the `Parapet.Integrations.Scoria` telemetry adapter to safely consume `Scoria.SRE.Telemetry` events, translating them into Parapet metrics and creating durable incidents for AI tool failures.

Purpose: Bridges the Scoria AI library with Parapet's observability stack while enforcing strict cardinality limits.
Output: The `Parapet.Integrations.Scoria` module and its tests.
</objective>

  <name>Scoria Telemetry Adapter</name>

  <behavior>
    - Test 1: `setup/0` attaches handler to `[:scoria, :sre, :telemetry]` without error.
    - Test 2: `handle_event/4` translates successful events to `[:parapet, :scoria, :metrics]`, explicitly extracting and keeping only low cardinality labels (`model`, `provider`, `tool_name`).
    - Test 3: `handle_event/4` computes an `:outcome` key (`:success` or `:failure`) based on whether `:error` is present.
    - Test 4: `handle_event/4` routes errors to `Parapet.Evidence.create_incident/1`.
    - Test 5: `handle_event/4` rescues any exceptions during processing to prevent crashing the host process.
  </behavior>
  <implementation>
    Implement `Parapet.Integrations.Scoria` following the pattern in `Parapet.Integrations.Accrue`.
    - Use `:telemetry.attach/4` in `setup/0`.
    - In `handle_event/4`, filter `metadata` with `Map.take/2` for safe labels (e.g. `:model`, `:provider`, `:tool_name`).
    - Calculate `:outcome` as `:success` or `:failure` based on the `:error` key presence.
    - Emit the translated event using `:telemetry.execute/3`.
    - Invoke `Parapet.Evidence.create_incident/1` if the metadata contains an `:error`, formatting the description with `inspect(metadata[:error])`.
  </implementation>
</feature>
