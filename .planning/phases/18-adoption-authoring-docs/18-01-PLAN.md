<objective>
Make integration activation uniform and crash-proof in code so all four integration guides can honestly show the same `Parapet.attach(adapters: [...])` line (D-16, resolving OQ-1 → option (b)+(d)). This SUPERSEDES D-07's "Rulestead guide shows `Parapet.Integrations.Rulestead.attach()` directly" instruction.

Purpose: `Parapet.attach(adapters: [:rulestead])` currently raises `UndefinedFunctionError` because Rulestead is the sole integration exposing only `attach/0` (not `setup/0`), while `Parapet.attach/1` hard-codes `apply(module, :setup, [])`. This is a live crash in the 30-minute adopter path. Introducing a `Parapet.Integration` behaviour also turns this whole bug class into a compile error for any future integration.

Output: A `Parapet.Integration` behaviour, `@behaviour` + `@impl true` declarations on all eight integration modules, a `setup/0` delegate on Rulestead, a corrected `Parapet.attach/1` `@doc`, a CHANGELOG `0.10.0` entry, and tests proving Rulestead now activates via the uniform line.
</objective>
