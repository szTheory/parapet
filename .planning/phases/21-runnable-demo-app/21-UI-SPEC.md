# Phase 21 тАФ UI Design Contract

> Visual and interaction contract for Phase 21: Runnable Demo App.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Context Note

Phase 21 is an Elixir/Phoenix library project тАФ **not a React/Next.js/Vite project**. The shadcn gate does not apply. No design system tooling (shadcn, Radix, etc.) is used. The Operator UI is a Phoenix LiveView generated by `mix parapet.gen.ui` from committed `.eex` templates. The design contract for this phase is a **fidelity contract**: the demo app must serve the existing Operator UI templates with Tailwind CSS compiled and active so they render correctly for human visitors.

The visual design is pre-determined by:
- `priv/templates/parapet.gen.ui/operator_components.ex.eex`
- `priv/templates/parapet.gen.ui/operator_live.ex.eex`
- `priv/templates/parapet.gen.ui/operator_detail_live.ex.eex`

No new visual decisions are required beyond configuring Tailwind correctly. The executor must NOT deviate from the committed template output.

## Spacing Scale

Declared values (Tailwind 8-point scale, multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`p-1`, `gap-1`) | Icon gaps, inline badge padding |
| sm | 8px (`p-2`, `gap-2`, `px-2 py-1`) | Badge padding, compact element spacing |
| md | 16px (`p-4`, `gap-4`, `px-4 py-3`) | Card padding, list row padding |
| lg | 24px (`p-6`, `mb-6`) | Section padding, component bottom margins |
| xl | 32px (`gap-8`) | Layout column gaps (queue + detail split) |
| 2xl | 48px | Major section breaks (page-level) |
| 3xl | 64px | Page-level outer padding |

Exceptions:
- Timeline connector line uses `pb-8` (32px) for spacing between entries тАФ non-standard but matches template exactly
- Badge font size exception: `text-[10px]` used for status/actor badges in timeline and escalation chain тАФ smaller than the 14px minimum to match template output; do not change
- Touch targets on mobile (back navigation) should be minimum 44px tall

Source: Template inspection of `operator_components.ex.eex` тАФ spacing classes extracted directly.

## Typography

All sizes come directly from the existing Tailwind classes in the committed templates. Do not override.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / list item text | 14px (`text-sm`) | 400 regular | 1.5 (Tailwind default) |
| Label / secondary text | 12px (`text-xs`) | 400 regular | 1.4 |
| Heading (incident title) | 24px (`text-2xl`) | 700 bold | 1.2 |

## Color

The Operator UI templates use a **stone/gray neutral base** with amber for escalation/warnings, blue for impact/changes, and indigo for trace links. Colors are Tailwind utility classes тАФ no custom CSS variables or hex values defined.

| Role | Tailwind Value | Usage |
|------|----------------|-------|
| Dominant surface (60%) | `bg-white` / `stone-50` / `gray-50` | Page background, card backgrounds, list row default |
| Secondary surfaces (30%) | `stone-200` (dividers), `gray-200` (borders) | Card borders, dividers, section separators |
| Accent тАФ state badges (10%) | See state color map below | Incident state pills, severity badges only |
| Warning/escalation | `amber-50` bg + `amber-950` text + `amber-200` border | Escalation section, warning banners |
| Impact info | `blue-50` bg + `blue-900` text + `blue-200` border | Impact summary cards, recent system changes |
| Destructive | `red-*` (state badge for critical severity) | Destructive state indicators only |

**State color mapping** (from `state_color/1` function implied in templates):
- `"open"` тЖТ `bg-red-100 text-red-800` (attention-critical)
- `"investigating"` тЖТ `bg-amber-100 text-amber-800` (in-progress)
- `"resolved"` тЖТ `bg-green-100 text-green-800` (success)

**Accent reserved for:**
- Incident state pills (`open`, `investigating`, `resolved`)
- Severity badge pills
- Timeline entry badge circles (actor class color)
- Escalation chain step status badges
- Selected queue row left border accent

The trace link uses `text-indigo-700 hover:underline` тАФ reserved for external navigation links only.

Source: `operator_components.ex.eex` classes extracted across all component functions.

## Tailwind Build Requirement

The demo app must include a working Tailwind build to render the Operator UI correctly for human visitors. The smoke test (GET /parapet = 200) passes without Tailwind тАФ but the human-facing demo goal requires styled output.

**Required Tailwind configuration in `examples/demo_app/`:**

```
assets/
  css/
    app.css         # @import "tailwindcss/base"; @import "tailwindcss/components"; @import "tailwindcss/utilities";
  tailwind.config.js  # content: ["./lib/**/*.{ex,eex,heex}", "../priv/templates/**/*.eex"]
```

The `content` glob in `tailwind.config.js` **must include** `"../../priv/templates/parapet.gen.ui/*.eex"` (relative to the demo app's assets directory, resolving to the parapet library's template files) so that all Tailwind classes used in the committed generated LiveView files are included in the purged CSS bundle.

Alternatively, set `content` to the committed generated files directly: `"./lib/demo_app_web/live/parapet/**/*.ex"`.

Source: RESEARCH.md Standard Stack note on Tailwind/heroicons; template class inspection.
