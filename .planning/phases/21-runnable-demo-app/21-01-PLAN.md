CRITICAL MIGRATION FACT: `mix parapet.gen.spine` generates a migration body that creates
the spine tables but does NOT add `runbook_data` or `trace_id` columns to
parapet_incidents. In the parapet library itself those columns are added by SEPARATE
later migrations (priv/repo/migrations/20260511000000_add_runbook_data_to_incidents.exs
and 20260516233447_add_trace_id_to_incidents.exs). The demo app does not get those.
The Incident schema (lib/parapet/spine/incident.ex:47,55) casts runbook_data and trace_id,
and the demo seeds (Plan 03) write runbook_data (DEMO-02 warning step). Therefore the
demo's single committed migration MUST include both columns or the seeds will fail with
an "undefined column" error at insert time.
