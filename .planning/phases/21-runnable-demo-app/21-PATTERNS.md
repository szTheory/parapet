```elixir
defmodule DemoApp.Application do

  use Application

  @impl true
  def start(_type, _args) do
    children = [
      DemoApp.Repo,
      {Phoenix.PubSub, name: DemoApp.PubSub},
```

Key difference from parapet's Application: demo app starts a Repo, PubSub, and Endpoint rather than `Parapet.Capabilities`.

### `examples/demo_app/priv/repo/migrations/<ts>_add_parapet_spine_tables.exs` (migration, CRUD)

**Analog:** `lib/mix/tasks/parapet.gen.spine.ex` lines 25-89 — the migration body is embedded verbatim in the generator.

**Complete migration body** (from `parapet.gen.spine.ex` lines 27-88):
```elixir
def change do
  create table(:parapet_action_items, primary_key: false) do
    add :id, :binary_id, primary_key: true
    add :title, :string, null: false
    add :integration, :string, null: false
    add :external_id, :string, null: false
    add :state, :string, default: "open", null: false
    timestamps()
  end

  create table(:parapet_incidents, primary_key: false) do
    add :id, :binary_id, primary_key: true
    add :state, :string, default: "open", null: false
    add :title, :string, null: false
    add :description, :text
    add :correlation_key, :string
    timestamps()
  end

  create unique_index(:parapet_incidents, [:correlation_key], where: "state = 'open'")
  create index(:parapet_incidents, [:updated_at, :id], where: "state in ('open', 'investigating')")
  create index(:parapet_incidents, [:updated_at, :id], where: "state = 'resolved'")

  create table(:parapet_timeline_entries, primary_key: false) do
    add :id, :binary_id, primary_key: true
    add :type, :string, null: false
    add :payload, :map, default: %{}
    add :incident_id, references(:parapet_incidents, type: :binary_id, on_delete: :delete_all), null: false
    timestamps()
  end

  create index(:parapet_timeline_entries, [:incident_id])
  create index(:parapet_timeline_entries, [:incident_id, :inserted_at])

  create table(:parapet_tool_audits, primary_key: false) do
    add :id, :binary_id, primary_key: true
    add :tool_name, :string, null: false
    add :input, :map, default: %{}
    add :output, :map, default: %{}
    add :success, :boolean, default: false, null: false
    add :duration_ms, :integer
    add :timeline_entry_id, references(:parapet_timeline_entries, type: :binary_id, on_delete: :delete_all)
    timestamps()
  end

  create index(:parapet_tool_audits, [:timeline_entry_id])
  create index(:parapet_tool_audits, [:timeline_entry_id, :inserted_at])

  create table(:parapet_system_events, primary_key: false) do
    add :id, :binary_id, primary_key: true
    add :type, :string, null: false
    add :payload, :map, default: %{}
    timestamps()
  end

  create index(:parapet_system_events, [:inserted_at])
end
```

NOTE: The migration does NOT include `trace_id` column on `parapet_incidents` even though `Parapet.Spine.Incident` schema has a `trace_id` field. This is the actual generator output as of the current codebase. Run `mix parapet.gen.spine` (or `mix parapet.install`) to produce the timestamped migration file — commit that output directly. Do not manually add `trace_id`; match exactly what the generator produces.

Also note: `runbook_data` column is absent from the migration body above. The `Incident` schema field `runbook_data: :map` must exist in the DB for seeds to write it. This is a discrepancy to investigate — run the generator and inspect actual output before committing.

```elixir
{:ok, incident} = Parapet.Evidence.create_incident(%{
  title: "Login service elevated error rate",
  description: "Auth endpoint returning 5xx",
  state: "open",
  correlation_key: "login-error-rate-spike",
  runbook_data: %{
    "title" => "Login Failure Runbook",
    "steps" => [
      %{
        "id" => "check_metrics",
        "label" => "Check metrics dashboard",
        "description" => "Verify DB connection pool and error distribution",
        "type" => "manual",
        "kind" => "guidance",
        "warning" => "High cardinality risk — check for label explosion before querying Prometheus",
        "guidance" => nil,
        "requires_preview" => false,
        "preview_only" => false,
        "auto_execute" => false
      }
    ]
  }
})

{:ok, _} = Parapet.Evidence.append_timeline(incident.id, %{
  type: "note",
  payload: %{"text" => "Alert triggered — investigating"}
})

{:ok, _} = Parapet.Evidence.log_tool_audit(%{

  output: %{"status" => "ok"},
  success: true,
  duration_ms: 23
})
```

**CRITICAL — string keys in `runbook_data`:** The `runbook_data` field is stored as Ecto `:map` (JSONB). All keys become strings on DB round-trip. Always use string keys in seed data (see `evidence.ex` line 55: `field(:runbook_data, :map)`).

**CRITICAL — valid Incident fields only:** Cast fields are `title`, `description`, `state`, `correlation_key`, `trace_id`, `runbook_data` (from `spine/incident.ex` lines 53-58). `source` and `alert_name` do NOT exist.

Key CI notes:
- `demo` depends on `test` via `needs: [test]` — only runs when main test suite passes
- `demo` uses explicit postgres:16-alpine service container (main `test` job relies on runner's pre-installed postgres — no service block there)
- No `continue-on-error` on either new job
- Cache key uses `examples/demo_app/mix.lock` (NOT `**/mix.lock`) to avoid cache collisions with root deps
