# v0.6 Requirements: Change Correlation & Audit Trailing

## Overview
Finding the *cause* of an incident is often the most time-consuming part of SRE triage. Changes (deployments or feature flag flips) are the root cause of the vast majority of regressions. Milestone v0.6 focuses on integrating deeply with `Rulestead` to correlate flag changes with SLO burn rates, pulling OpenTelemetry trace exemplars for deep linking, and bridging Parapet's internal tool audits with `Threadline` for compliance.

## System Requirements

### RUL-01: Rulestead Flag Telemetry
- [ ] System consumes `Rulestead` telemetry events for feature flag toggles.
- [ ] System durably records flag toggles as `Incident` timeline entries if the flag targets a significant user population or global state.
- [ ] System emits `parapet_rulestead_flag_change_total` metrics with labels for the flag name and new state to overlay on Grafana dashboards.

### RUL-02: Operator UI Flag Correlation
- [ ] System Operator UI highlights any Rulestead flags toggled within a configurable window (e.g., 15 minutes) prior to an incident's creation.
- [ ] System allows operators to view the precise state change of the flag from the Incident timeline.

### OTL-01: OpenTelemetry Trace Exemplars
- [x] System telemetry ingestion supports extracting OpenTelemetry `trace_id` and `span_id` from event metadata or the current process dictionary.
- [ ] System attaches extracted trace IDs to generated Prometheus metrics as exemplars to allow Grafana to natively link from metrics to traces.
- [ ] System records `trace_id` on durable `Incident` records when an alert triggers, allowing the UI to link directly to the trace backend (e.g., Jaeger, Tempo, Honeycomb) via a configurable MFA template.

### THR-01: Threadline Compliance Sync
- [ ] System `Parapet.Ecto.ToolAudit` can be configured to broadcast audit events to `Threadline` (if available).
- [ ] System allows operators to defer the storage of audit logs entirely to `Threadline` in compliance-heavy environments to maintain a single source of truth for administrative actions.

## Acceptance Criteria
- [ ] A developer can toggle a Rulestead flag and see a deploy/change marker in Grafana.
- [ ] An active incident in the Operator UI clearly flags the toggled Rulestead flag as a potential cause.
- [ ] A Parapet-generated Prometheus metric includes `TraceID` exemplars.
- [ ] A `ToolAudit` record successfully mirrors to a `Threadline` event log without breaking the "compile out cleanly" constraint.
