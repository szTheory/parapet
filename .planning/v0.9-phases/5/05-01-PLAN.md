<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Oban worker process -> Postgres claim row | Untrusted concurrent executors compete for the same logical action |
| Test process swarm -> SQL sandbox owner | Concurrent proof tests must share DB ownership without hiding lock behavior |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | Action ownership row | mitigate | Enforce a unique DB claim key and explicit lifecycle transitions in `parapet_action_claims`. |
| T-05-02 | Denial of Service | Claim transaction window | mitigate | Keep the transaction short, claim first, and push external effects outside the lock window per D-24. |
| T-05-03 | Repudiation | Concurrency loser outcomes | mitigate | Persist explicit loser or short-circuit states so operators and tests can prove why a duplicate did not run. |
</threat_model>
