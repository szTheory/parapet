| Background updates -> visible queue | Realtime awareness must not silently reorder the operator’s current reading context |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Access Control | Generated queue param handling | mitigate | Only pass bounded validated params to the public operator API and keep the LiveView inside the host-owned authenticated mount pattern. |
| T-03-05 | Denial of Service | Generated queue mount/render path | mitigate | Remove full-queue mount fetches and stream only the current page. |
| T-03-06 | Tampering / UX safety | Background queue updates | mitigate | Surface an explicit refresh affordance and avoid silent stream inserts that reorder the operator’s visible queue. |
</threat_model>
